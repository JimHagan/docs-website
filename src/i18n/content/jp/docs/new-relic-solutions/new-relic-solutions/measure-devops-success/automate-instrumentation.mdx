手動で行っていた装置を自動化することで、さまざまなメリットが得られます。自動化することでスピードアップし、ファットフィンガーエラーをなくすことができ、さらに重要なのは観測性を向上させることです。システムのインスツルメンテーションに費やす時間が減り、開発エコシステムの拡大に伴う苦労が軽減されます。

## 前提条件

このチュートリアルを始める前に、 [Measure code pipeline](/docs/using-new-relic/welcome-new-relic/measure-devops-success/measure-code-pipelines) チュートリアルを完了しておいてください。

## 1.インストルメンテーションをすべてのビルドに組み込む [#make-instrumentation-build]

標準的なビルドプロセスにインスツルメンテーションを組み込むことで、開発者にとって新たな負担になるのではなく、可視性がデフォルトになります。 [Gradle](https://gradle.org/) のような最新のビルドツールは、ほとんど何でもできます。その力を利用して、コードを迅速かつ効率的にインスツルメンテーションすることができます。

<Callout variant="tip">
  パイプラインにおけるインスツルメンテーションの自動化については、この [Best Practices の記事](https://blog.newrelic.com/engineering/best-practices-automating-monitoring/) をお読みください。
</Callout>

Terraformのような構成管理ツールでも、自動化が可能です。 [New Relic Terraform Providerを使用すると、](https://github.com/newrelic/terraform-provider-newrelic) エージェントやアラートを自動的に追加したり、新しいサービスを立ち上げる際に [ダッシュボード](/docs/query-your-data/explore-query-data/dashboards/introduction-dashboards) を作成したりすることができます。

![New Relic terraform snippet](./images/new-relic-terraform-snippet.png "new-relic-terraform-snippet.png")

## 2.CLIツールの活用 [#cli-tools]

ほとんどのツールには、作業を素早く行うための裏技的なコマンドラインインターフェイスが用意されています。例えば、 [New Relic CLI](https://github.com/newrelic/newrelic-cli) では、アカウント間でのエンティティの検索やタグ付けができます。 [New Relic Lambda CLI](https://github.com/newrelic/newrelic-lambda-cli) では、AWS Lambda 関数に New Relic の AWS Lambda レイヤーをインストールして設定します。また、 [NR-1 CLI](https://developer.newrelic.com/explore-docs/nr1-cli) を使用すると、独自の New Relic アプリケーションを構築して共有することができます。

## 3.APIの力を活用する [#leverage-apis]

サービスのインスツルメンテーションが完了したら、New Relic の REST API を利用して、インスツルメンテーションから情報を取得したり、監視プロセスを管理したりすることができます。 [New Relic API Explorer](/docs/apm/apis/api-explorer-v2/getting-started-new-relics-api-explorer) では、利用可能な各APIコールのcURLリクエストフォーマット、利用可能なパラメーター、潜在的なレスポンスステータスコード、JSONレスポンス構造などを確認することができます。

* [シンセティック・モニターを定義するためのAPI](/docs/apis/synthetics-rest-api/monitor-examples/manage-synthetics-monitors-rest-api)
