## 概要

New Relic は、オープンソースの Apache Log4j フレームワークに存在する重要な脆弱性 CVE 2021-44228 および CVE-2021-45046 に対処するため、Containerized Private Minion (CPM) [バージョン 3.0.57](/docs/release-notes/synthetics-release-notes/containerized-private-minions-release-notes/containerized-private-minion-cpm-3057/) を 2021-12-14 にリリースしました。悪意のある行為者は、ログメッセージまたはログメッセージのパラメーターを使用して任意のコードを実行できる可能性があります。

また、New Relic は、CVE-2021-44228 および CVE-2021-45046 に対処するため、2021-12-14 に Helm Charts バージョン 1.0.45 をリリースしました。Helm Charts バージョン 1.0.45 には、CPM バージョン 3.0.57 が含まれています。

ニューレリックは、新しい情報が得られ次第、このセキュリティブルテンおよびお客様向けガイダンスを更新します。

**脆弱性の識別子:** NR21-04

**プライオリティ：** クリティカル

Helm Charts を使用して CPM の設定を更新している場合は、New Relic Helm Charts バージョン 1.0.45 を導入する必要があります。これにより、CPMがバージョン3.0.57にアップデートされます。

## アクションアイテム [#action]

New Relic Containerized Private Minion の CVE-2021-44228 を修正するため、お客様にはできるだけ早くバージョン 3.0.57 にアップグレードすることをお勧めします。このバージョンは、修復された Apache Log4j フレームワークの 2.16.0 バージョンを使用するように更新されています。CPM の更新は、Helm Charts バージョン 1.0.45 から行うことができます。

<Callout variant="important">
  すでに Containerized Private Minion (CPM) [バージョン 3.0.55](/docs/release-notes/synthetics-release-notes/containerized-private-minions-release-notes/containerized-private-minion-cpm-3055/) にアップグレードしている場合、Apache Foundation の現在の [技術ガイダンス](https://logging.apache.org/log4j/2.x/security.html#CVE-2021-44228) では、使用している CPM は CVE 2021-44228 に対する保護が不完全で、リモートでコードを実行される可能性が残っています。現時点では、CPM バージョン 3.0.57 へのアップデートを強く推奨します。
</Callout>

このステップでは、New Relic Containerized Private Minion (CPM)のみを修復します。また、New Relic の Java エージェントを更新する必要があるかもしれません。詳細については、 [NR21-03](/docs/security/new-relic-security/security-bulletins/security-bulletin-nr21-03) を参照してください。

**アプリケーションにlog4jを直接使用しているお客様は、 [Apache Log4j Security Vulnerabilities](https://logging.apache.org/log4j/2.x/security.html) ページをよくご覧になり、考慮すべき改善策をご確認ください。**

## 脆弱性情報 [#vuln-info]

2021-12-09、log4j フレームワークに高レベルの脆弱性が公開されました。攻撃者は、ログメッセージやログメッセージのパラメータを使って任意のコードを実行することができます。

* [CVE-2021-44228 cvss 10.0](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2021-44228)
* [CVE-2021-45046 cvss 9.0](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2021-45046)
* [Apache log4jのセキュリティの脆弱性](https://logging.apache.org/log4j/2.x/security.html)
* [New Relic Explorers Hub](https://discuss.newrelic.com/t/log4j-zero-day-vulnerability-and-the-new-relic-java-agent/170322)
