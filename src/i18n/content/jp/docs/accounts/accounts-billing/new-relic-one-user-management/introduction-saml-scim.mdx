New Relic ユーザーの New Relic への追加方法、管理方法、ログイン方法などを自動的に制御するために、以下の機能を提供しています。

* **SAML SSO**: これにより、ユーザーはNew Relicへのログインに、デフォルトのEメール＋パスワードではなく、シングルサインオン（SSO）のIDプロバイダーサービスを使用することができます。

  * 要件： [Pro または Enterprise エディション](https://newrelic.com/pricing).

* **SCIM プロビジョニング** 、AUM（Automated User Management）とも呼ばれています。SCIM プロビジョニングにより、組織はアイデンティティプロバイダーサービスを使用して、New Relic でのユーザーの追加や更新を自動化することができます。定義上、SCIM プロビジョニングには SAML SSO の使用が必要ですが、SAML SSO には SCIM は必要ありません。

  * 要件： [Enterprise edition](https://newrelic.com/pricing) 、組織が New Relic One のユーザーモデルになっていること（詳細は後述します）。

これらを有効にする前に、これらの機能に影響を与えるもう一つの大きな要因、つまりあなたの組織がどのユーザーモデルを使っているかを理解することが重要です。2020年、私たちはNew Relic Oneユーザーモデルと呼ばれる、新しく改良されたユーザーモデルをリリースしました。当初は主に新規のお客様がこのモデルを利用していましたが、現在では [のユーザー移行手順により、一部の古いお客様にもご利用いただけるようになっています](/docs/accounts/original-accounts-billing/original-users-roles/user-migration/) 。順次、すべてのユーザーがこのモデルに移行される予定です。

SAMLおよびSCIMを有効にする方法については、以下のユーザーモデルを選択してください。自分のユーザーモデルがわからない場合は、 [Overview of user model](/docs/accounts/original-accounts-billing/original-product-based-pricing/overview-changes-pricing-user-model/#user-models) を参照してください。
