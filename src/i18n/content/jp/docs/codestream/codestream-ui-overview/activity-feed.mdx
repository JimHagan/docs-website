アクティビティフィードは、チームメイトが投稿した新しいコメント、課題、フィードバックリクエストや、既存のディスカッションへの新しい返信などを知ることができる決定的な場所です。アクティビティフィードを表示していないときでも、アクティビティフィードアイコンのバッジを見れば、新しい情報があるかどうかがわかります。青いバッジに白い点がついている場合は、新しいディスカッションや返信があることを意味します。青いバッジの中に数字が入っている場合は、あなたが言及された新しいディスカッションや返信があることを意味します。

![アクティビティフィードバッジ](/images/ActivityFeedWithBadge3.png)

IDEのステータスバーにあるCodeStreamのセクションでは、フィードに新しいメッセージがあるかどうかを知ることができます。ユーザー名の右にあるドットは新しいメッセージがあることを意味し、数字は新しい言及があることを意味します。

![ステータスバー](/images/StatusBarWithMentions2.png)

アクティビティフィードの上部にあるフィルターは、フィードに表示される内容をコントロールします。

* **自分の組織の全員からのアクティビティ**: 自分の組織からのすべてのアクティビティを見ることができます。小規模な組織にのみお勧めします。
* **自分のIDEで開いているコードに関連するアクティビティ（デフォルト）**: 自分のIDEで開いているリポジトリのコードに関連するディスカッションを見る。
* **Activity associated with code in selected folder**: チームがmonorepos内で作業している場合、このオプションを使用します。現在選択されているフォルダ内のコードに関連するディスカッションを見ることができます。

フィルターの設定にかかわらず、あなたが@mentionedされたときには常に通知されます。

![アクティビティフィードフィルター](/images/ActivityFeedFilters.png)

予想通り、新しいものはフィードの一番上に追加されます。それが非常に古い議論に対する新しい返信であってもです。CodeStreamでは、新しいアクティビティの左側に青いボーダーを追加して、フラグを立てています。例えば、これはフィードの一番上にある新しいコメントです。

![フィードの新しいコメント](/images/ActivityFeed-NewCodemark.png)

ここには、コメントに対する新しい返信があります。コメントはすでに見られているので、返信だけが青い枠で表示されています。

![フィードでの新しい返信](/images/ActivityFeed-NewOldReplies.png)

**See 1 earlier reply** は、すでに見たことのある以前の返信があることを意味しています。メインのアクティビティフィードビューには、新しい未読の返信のみが表示されますが、このリンクをクリックすると、ディスカッションスレッド全体を見ることができ、新しい返信を追加することができます。

![拡大コメント](/images/CodemarkViewWithReplies1.png)

アクティビティフィードは、チームの時間の始まりまでさかのぼりますが、特定の古いコードマークを探している場合は、 [フィルターや検索](/docs/codestream/codestream-ui-overview/filter-search) ツールを使用するとよいでしょう。
